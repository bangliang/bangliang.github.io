Java-List

List 是用来管理对象的集合接口，代表的是有序的队列。

List有以下几种:

1.AbstractList 略

2.AbstractSequentialList 略

3.ArrayList：是一个数组队列，相当于动态数组。由数组实现，线程不同步（非线程安全的）。

随机访问率高，随机插入、删除效率低。

4.LinkedList：是双向链表，查询慢，增删快。

5.Vector：是矢量队列，与ArrayList一样，是一个动态数组，由数组实现。

6.Stack：是栈，继承于Vector，先进后出。

使用场景

1.快速插入、删除元素：LinkList

2.快速访问元素：ArrayList

3.单线程或仅被单线程操作：ArrayList

4.多线程或被多个线程操作 ：Vector

LinkedList和ArrayList的区别

1.在插入元素时，由于LinkedList是双向链表，在查找要插入节点的位置index时，会有一个加速动作：若inde<链表长度1/2，向前查找；若>，向后查找。而ArrayList在插入时，会移动index后的所有元素。

2.在访问元素时，ArrayList可以直接返回数组中index位置的元素，而LinkedList则是与插入元素时寻找index位置相同，需要进行查找。

LinkedList和Vector的区别

1.LinkedList适用于单线程，Vector适用于多线程。

2.LinkedList支持序列化，Vector不支持。

3.Vector比LinkedList多一个指定容量增加系数的构造函数.

4.容量增加方式不同。

5.Vector支持Enumeration去遍历，LinkedList不支持。

hexo安装和部署

1.安装Git Bash

2.安装Node.Js

3.安装Hexo
· 先新建一个文件夹，在文件夹内右键点击“Git Bash Here”或者是“在此次打开PowerShell窗口”
· 安装Hexo命令: npm i -g hexo
· 查看版本命令: hexo -v
· 初始化命令: hexo init，然后可以看到文件夹内生成了一些文件

4.github本地环境配置，配置好了可以忽略

5.github上新建一个repo，名称为***.github.io

6.修改新建文件夹内名为_config.yml的文件的配置，注意，冒号后面都有一个半角空格
deploy:
type: git
repo: https://github.com/YourgithubName/YourgithubName.github.io.git
branch: master

7.继续敲命令行，依次执行下列命令
hexo clean
hexo generate
hexo server
打开浏览器输入：http://localhost:4000，就可以看到了

8.上传到github
· 安装：npm install hexo-deployer-git –save
· 执行下列命令:
hexo clean
hexo generate
hexo deploy

9.我在安装过程中遇到的bug
· 安装Hexo时，提示“no longer maintained”，npm换源即可解决
· 在执行“hexo deploy”时，提示“bash：/dev/tty：No such device or address”，检查一些_config.yml配置文件有没有哪里写错。其中repo要填SSH Key。
